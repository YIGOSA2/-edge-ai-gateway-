# Edge AI Gateway

An edge AI intelligent gateway built on the RK3506 embedded Linux platform, integrating multi-sensor data acquisition, on-device AI anomaly detection, local SQLite storage, MQTT cloud reporting, and a real-time web dashboard.

## System Architecture

```
┌──────────────────────────────────────────────────────────┐
│                         main.c                            │
│  Init sensors → Spawn 4 threads → Wait → Cleanup & exit  │
└─────┬──────────┬──────────┬──────────┬───────────────────┘
      │          │          │          │
 ┌────▼───┐ ┌───▼────┐ ┌───▼────┐ ┌───▼────┐
 │Sensor  │ │Storage │ │  AI    │ │ MQTT   │
 │Thread  │ │Thread  │ │Thread  │ │Thread  │
 │100ms   │ │ 1s     │ │ 1s     │ │ 5s     │
 └────┬───┘ └───┬────┘ └───┬────┘ └───┬────┘
      │          │          │          │
 ┌────▼──────────▼──────────▼──────────▼────┐
 │        shared_data_t (mutex-protected)    │
 │  data: sensor_data_t (latest readings)    │
 │  ai_status: AI result (0=normal, 1=alert) │
 └───────────────────────────────────────────┘
      │                     │                │
 ┌────▼────┐         ┌─────▼─────┐    ┌─────▼──────┐
 │AHT30    │         │SQLite DB  │    │MQTT Broker │
 │MPU6050  │         │sensor.db  │    │  (cloud)    │
 │(I2C)    │         │(local)    │    └──────┬─────┘
 └─────────┘         └───────────┘           │
                                       ┌─────▼──────┐
                                       │  Web Dash  │
                                       │ dashboard  │
                                       └────────────┘
```

## Hardware

| Component | Model | Interface | I2C Address |
|-----------|-------|-----------|-------------|
| Dev Board | RK3506EVM | — | — |
| Temperature & Humidity | AHT30 | I2C | 0x38 |
| Accelerometer | MPU6050 / MPU6500 | I2C | 0x68 |

## Features

- **Multi-sensor acquisition** — AHT30 temperature/humidity + MPU6050 3-axis acceleration, 100ms sampling
- **On-device AI inference** — Dual-path anomaly detection: sliding-window variance for vibration + Random Forest rules for environmental thresholds
- **Local SQLite storage** — Persistent storage of all sensor readings at 1-second intervals
- **MQTT cloud reporting** — JSON telemetry published to cloud broker every 5 seconds
- **Real-time web dashboard** — Pure frontend dashboard via MQTT WebSocket with flashing anomaly alerts
- **Multi-threaded architecture** — Sensor/Storage/AI/MQTT threads decoupled with mutex-protected shared data

## Quick Start

### Prerequisites

- **Cross-compilation toolchain**: `arm-buildroot-linux-gnueabihf-gcc` (Buildroot SDK)
- **Target board dependencies**: `libpthread`, `libsqlite3`, `mosquitto_pub`
- **Python training environment**: `numpy`, `pandas`, `scikit-learn`

### Build & Deploy

```bash
cd src

# Edit Makefile to point to your Buildroot SDK
# SDK_DIR = /path/to/arm-buildroot-linux-gnueabihf_sdk-buildroot

# Build
make clean && make

# Deploy to the board (edit BOARD_IP in Makefile)
make deploy

# Run on the board
./gateway

# Graceful exit (Ctrl+C)
```

### AI Model Training (Optional)

```bash
cd ai

# 1. Label data
python3 label_data.py

# 2. Train model and export C inference code to src/ai_infer.h
python3 train_model.py
```

### Configuration

Before running, update the MQTT broker address in:

- `src/mqtt_thread.c` — `MQTT_BROKER_IP` macro
- `web/dashboard.html` — `BROKER` variable

## AI Anomaly Detection

A dual-path detection strategy covers different anomaly scenarios:

| Detection Path | Method | Trigger Condition |
|----------------|--------|-------------------|
| Vibration | 5-sample sliding window variance | Combined 3-axis variance > 1,000,000 |
| Environmental | Random Forest exported rules | Temperature > 35.05°C or Humidity > 65% |

An alarm is only triggered after **3 consecutive** anomaly detections to reduce false positives.

Training data: 2,696 samples (2,493 normal, 203 anomaly), model accuracy 99.8%.

## Web Dashboard

Open `web/dashboard.html` in a browser. It connects to the MQTT broker via WebSocket and displays:

- Temperature, humidity, and 3-axis vibration values
- AI detection status (normal / flashing red anomaly alert)
- Last 100 event log entries

> Note: The dashboard uses `ws://` connections. Access it via HTTP, not HTTPS.

## MQTT Protocol

```json
{
    "timestamp": 15112565,
    "temp": 27.72,
    "humi": 41.11,
    "vib_x": -11416,
    "vib_y": -112,
    "vib_z": 11792,
    "ai_status": 0,
    "ai_label": "normal"
}
```

- **Topic**: `edge-gateway/sensor`
- **QoS**: 1
- **Publish interval**: 5 seconds
